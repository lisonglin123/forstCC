# 性能优化 (Performance Optimization)

## 模型选择策略

**Haiku 4.5** (Sonnet 能力的 90%，节省 3 倍成本):
- 频繁调用的轻量级 agents
- 结对编程和代码生成
- 多 agent 系统中的 Worker agents

**Sonnet 4.5** (最佳编码模型):
- 主要开发工作
- 编排多 agent 工作流
- 复杂编码任务

**Opus 4.5** (最深层推理):
- 复杂架构决策
- 最大推理需求
- 研究和分析任务

## 上下文窗口管理

避免在以下情况使用最后 20% 的上下文窗口：
- 大规模重构
- 跨越多个文件的特性实施
- 调试复杂交互

较低上下文敏感性任务：
- 单文件编辑
- 独立实用程序创建
- 文档更新
- 简单 bug 修复

## Ultrathink + Plan Mode

对于需要深度推理的复杂任务：
1. 使用 `ultrathink` 增强思维
2. 启用 **Plan Mode** 以获得结构化方法
3. 通过多轮批判 "发动引擎 (Rev the engine)"
4. 使用拆分角色的子 agent 进行多样化分析

## 构建故障排除

如果构建失败：
1. 使用 **build-error-resolver** agent
2. 分析错误消息
3. 增量修复
4. 每次修复后验证
