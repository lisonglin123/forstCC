# /learn - 提取可重用模式 (Extract Reusable Patterns)

分析当前会话并提取任何值得保存为 skill 的模式。

## 触发条件

在会话期间解决了一个非平凡的问题后的任何时候运行 `/learn`。

## 要提取的内容

寻找：

1. **错误解决模式**
   - 发生了什么错误？
   - 根本原因是什么？
   - 怎么修复的？
   - 对类似的错误是否可重用？

2. **调试技术**
   - 不明显的调试步骤
   -有效的工具组合
   - 诊断模式

3. **变通方案 (Workarounds)**
   - 库的怪癖
   - API 限制
   - 特定版本的修复

4. **项目特定的模式**
   - 发现的代码库约定
   - 做出的架构决策
   - 集成模式

## 输出格式

在 `~/.claude/skills/learned/[pattern-name].md` 创建 skill 文件：

```markdown
# [描述性模式名称]

**Extracted:** [Date]
**Context:** [简要描述何时适用]

## Problem
[解决了什么问题 - 具体说明]

## Solution
[模式/技术/变通方案]

## Example
[代码示例（如果是适用）]

## When to Use
[触发条件 - 什么应该激活此 skill]
```

## 流程

1. 审查会话以寻找可提取的模式
2. 识别最有价值/可重用的见解
3. 起草 skill 文件
4. 保存前请求用户确认
5. 保存到 `~/.claude/skills/learned/`

## 说明

- 不要提取琐碎的修复（拼写错误、简单的语法错误）
- 不要提取一次性的问题（特定的 API 中断等）
- 专注于能节省未来会话时间的模式
- 保持 skills 专注 - 每个 skill 一个模式
